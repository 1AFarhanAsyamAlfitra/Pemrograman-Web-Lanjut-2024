1. APP_KEY dalam file setting .env Laravel adalah sebuah kunci yang digunakan untuk menghash data yang dapat diperlakukan oleh sistem pengamanan Laravel, seperti kunci untuk enkripsi dan dekripsi data.
2. Untuk men-generate nilai APP_KEY, Anda dapat menggunakan perintah php artisan key:generate di dalam terminal.
3. Secara default, Laravel memiliki 10 file migrasi. Migrasi adalah file yang digunakan untuk mengatur struktur dan data dalam database, dan diperlukan untuk mengatur tabel dan kolom yang dibutuhkan oleh aplikasi Anda.
4. Fungsi $table->timestamps(); di migrasi Laravel digunakan untuk menambahkan kolom created_at dan updated_at ke tabel, yang akan diisi secara otomatis dengan waktu saat data dibuat dan diperbarui.
5. Fungsi $table->id(); di migrasi Laravel digunakan untuk menambahkan kolom id dengan tipe data integer secara otomatis ke tabel.
6. Pada migrasi m_level, jika menggunakan $table->id();, maka kolom id akan dibuat dengan tipe data integer dan tidak akan memiliki nilai default. Jika menggunakan $table->id('level_id');, maka kolom level_id akan dibuat dengan tipe data integer dan memiliki nilai default 0.
7. Fungsi ->unique(); di migrasi Laravel digunakan untuk menambahkan konstrain unik pada kolom yang dibuat, yang akan memastikan bahwa nilai kolom tersebut tidak dapat duplikat.
8. Kolom level_id pada tabel m_user menggunakan $table->unsignedBigInteger('level_id'), sedangkan pada tabel m_level menggunakan $table->id('level_id') karena kolom level_id pada tabel m_user akan digunakan untuk membuat referensi ke tabel m_level, sementara kolom level_id pada tabel m_level akan digunakan untuk membuat referensi ke tabel m_user.
9. Kelas Hash digunakan untuk mengenkripsi data, seperti kata sandi. Fungsi Hash::make('1234') digunakan untuk mengenkripsi string "1234" menjadi kata sandi yang tidak dapat diketahui.
10. Tanda tanya (?) dalam query builder Laravel digunakan untuk menandakan bahwa nilai parameter akan diisi secara dinamis saat query dibangkitkan.
11. Pada Praktikum 6 - Tahap 3, tujuan penulisan kode protected $table = 'm_user'; dan protected $primaryKey = 'user_id'; adalah untuk mengatur nama tabel dan nama kolom primary key pada model Eloquent Laravel.
12. Menggunakan Query Builder, DB Façade, atau Eloquent ORM untuk melakukan operasi CRUD ke database (DB Façade / Query Builder / Eloquent ORM), Anda dapat memilih yang paling sesuai dengan kebutuhan Anda. Query Builder dan DB Façade lebih fleksibel dan dapat digunakan untuk melakukan query yang lebih kompleks, sementara Eloquent ORM lebih mudah digunakan untuk melakukan operasi CRUD pada tabel yang sudah didefinisikan dalam model.
